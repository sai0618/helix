# Grafana Cloud Observability

This project uses Grafana Cloud as the hosted Grafana UI. The dashboard is stored as code at:

```text
observability/grafana/dashboards/helix-gke-observability.json
```

The dashboard has three golden-signal panels backed by the BigQuery application log export:

- Traffic by URL.
- Errors by URL.
- Latency by URL.

It also has two Google Cloud Monitoring PromQL panels for the GKE `user-portal` workload:

- CPU usage for the `user-portal` container in the `helix` namespace.
- Memory usage for the `user-portal` container in the `helix` namespace.

The JSON uses Grafana dashboard schema v2. This is the format expected by Grafana's newer dashboard schema validation, where `annotations` is an array and panels live under `elements` and `layout`.

## Prerequisites

- BigQuery log export sinks are enabled in Terraform with `enable_bigquery_log_sinks = true`.
- Terraform grants BigQuery dataset read access to the Compute Engine default service account.
- The Compute Engine default service account can create BigQuery jobs in the Google Cloud Sandbox project.
- The Grafana Cloud stack has BigQuery and Google Cloud Monitoring datasources configured.

## Google Cloud Sandbox Service Account Key Setup

For Google Cloud Sandbox projects, use the Compute Engine default service account key for Grafana Cloud BigQuery. This works around the sandbox restriction that blocks custom project IAM grants such as `roles/bigquery.jobUser`.

The normal sandbox setup command creates this key automatically:

```text
credentials/compute-default-service-account-key.json
```

To create or replace the key manually:

```bash
scripts/create_grafana_bigquery_key.sh \
  --project-id PROJECT_ID \
  --force
```

The script:

- Resolves the project number.
- Uses the Compute Engine default service account: `PROJECT_NUMBER-compute@developer.gserviceaccount.com`.
- Reapplies the Terraform BigQuery dataset viewer bindings for the exported log datasets.
- Creates `credentials/compute-default-service-account-key.json`.
- Verifies BigQuery job creation with `SELECT 1 AS test`.
- Verifies read access to `helix_sandbox_user_portal_app_logs.stdout`.

Terraform grants that service account `roles/bigquery.dataViewer` on the exported log datasets through `log_export_viewer_members`.

In Grafana Cloud BigQuery datasource:

```text
Authentication type: Service Account Key
Service account key: contents of credentials/compute-default-service-account-key.json
Default project:     PROJECT_ID
Processing location: United States (US)
Service endpoint:    empty
Max bytes billed:    empty
```

Do not commit files under `credentials/`. The directory is ignored by Git.

## Forward OAuth Fallback

Forward OAuth Identity can work only when Grafana authenticates you through Google OAuth with the BigQuery scope. The BigQuery datasource cannot add this scope by itself.

If you use Forward OAuth Identity, datasource settings are:

```text
Authentication:      Forward OAuth Identity
Default project:     PROJECT_ID
Processing location: US
Service endpoint:    empty
```

Make sure Grafana Google OAuth includes this scope:

```text
https://www.googleapis.com/auth/bigquery
```

Do not paste OAuth scopes into the BigQuery SQL editor, dashboard query editor, or datasource query validation field. If Grafana shows an error like `failed to parse "openid email profile https://www.googleapis.com/auth/bigquery"`, that scope string is being executed as SQL and must be removed from the query field.

## Grafana Cloud Datasources

Create these datasources in Grafana Cloud:

| Datasource | Grafana plugin | Purpose |
| --- | --- | --- |
| BigQuery | BigQuery | Traffic, errors, and latency queries from exported app logs. |
| Google Cloud Monitoring | Google Cloud Monitoring | PromQL CPU and memory usage for the GKE `user-portal` workload. |

Use the same GCP project ID that Terraform writes into `iac/terraform/environments/sandbox/terraform.tfvars`.

For the Google Cloud Monitoring datasource, use the same service account key:

```text
Authentication type: Service Account Key
Service account key: contents of credentials/compute-default-service-account-key.json
Default project:     PROJECT_ID
```

## Import Dashboard

In Grafana Cloud:

1. Go to **Dashboards**.
2. Select **New**.
3. Select **Import** or the schema-v2 dashboard import option available in your Grafana Cloud UI.
4. Upload `observability/grafana/dashboards/helix-gke-observability.json`.
5. Select or set your existing datasources for:
   - `DS_BIGQUERY`
   - `DS_CLOUD_MONITORING`
6. Set dashboard variables if the project changes:
   - `project_id`
   - `app_log_dataset`

For each new Google Cloud Sandbox, set the defaults to the values generated by Terraform:

```text
project_id       = NEW_PROJECT_ID
app_log_dataset  = helix_sandbox_user_portal_app_logs
```

The dashboard query specs use BigQuery location `US` directly. Do not replace it with a dashboard variable unless your Grafana BigQuery plugin version supports variable interpolation in datasource query metadata.

## BigQuery Queries

The BigQuery SQL used by the dashboard is also documented in:

```text
docs/bigquery-grafana.md
```

Before importing the dashboard, verify the app log export table:

```bash
bq --project_id=PROJECT_ID ls helix_sandbox_user_portal_app_logs
```

Expected table:

```text
stdout
```

Run a sample query:

```bash
bq --project_id=PROJECT_ID query --use_legacy_sql=false \
  'SELECT timestamp, jsonPayload.message, jsonPayload.event_type, jsonPayload.version
   FROM `PROJECT_ID.helix_sandbox_user_portal_app_logs.stdout`
   ORDER BY timestamp DESC
   LIMIT 10'
```

## Notes

- If validation complains that `annotations` has an incorrect type, you are using a schema-v2 validator against a classic Grafana dashboard JSON. Use the checked-in dashboard file, which has `annotations` as an array.
- If validation complains that a panel query is missing `query`, `group`, or `version`, use the checked-in dashboard file. Each panel query is wrapped as a schema-v2 `DataQuery`.
- BigQuery log sink exports are asynchronous. New rows may take a few minutes to appear after logs reach Cloud Logging.
- BigQuery tables are created only after matching logs arrive.
- If the service account key says `bigquery.jobs.create` is missing, confirm the key is for the Compute Engine default service account, not `cli-service-account-1`.
- If the service account key cannot read `helix_sandbox_user_portal_app_logs.stdout`, rerun `./setup_new_sandbox.sh --project-id PROJECT_ID --skip-build --skip-deploy --skip-test` so Terraform reapplies log dataset viewer access.
- If Forward OAuth says `unauthorized`, use the service account key setup above unless your Grafana stack exposes Google OAuth scope configuration.
